The previous lessons defined the theory. This lesson makes it fully executable in Sonic Pi. Every Markov concept has a direct implementation using Sonic Pi's built-in tools — no external libraries needed.

## Sonic Pi's Stochastic Toolkit

```ruby
# choose — uniform random selection
note = [:c4, :e4, :g4, :b4].choose
# Each note has 25% probability. Every call returns one element, uniformly.

# rrand — continuous random in a range
amp_value = rrand(0.3, 0.8)
# Returns a float between 0.3 and 0.8 inclusive.

# rrand_i — integer random in a range
offset_beats = rrand_i(1, 4)
# Returns an integer: 1, 2, 3, or 4. Each equally likely.

# one_in(n) — true with probability 1/n
if one_in(4)
  play :c5, amp: 0.8  # this plays about 25% of the time
end
```

## Weighted Selection Using Array Repetition

Sonic Pi's `wrand` method takes weights as a separate array:

```ruby
pitches = [:c4, :e4, :g4, :b4]
weights = [4,   2,   2,   1]   # c4 is 4× more likely than b4

next_note = pitches.wrand(weights)
```

The weights don't need to sum to any specific value — they're relative. `[4, 2, 2, 1]` is equivalent to `[40, 20, 20, 10]`. This makes it easy to reason about relative likelihoods without calculating exact percentages.

## Assembling a Complete Character Voice

Here is a complete character voice using all the tools together. This character is dense and insistent, favouring the minor second interval, rarely resting, moving mostly by half and whole steps:

```ruby
# A dense, insistent character — prefers half-step pressure, rarely rests
insistent_transitions = {
  :e4 => { pitches: [:e4, :f4, :f4, :d4, :g4],     weights: [3, 4, 2, 2, 1] },
  :f4 => { pitches: [:e4, :e4, :f4, :g4, :eb4],     weights: [4, 2, 2, 2, 1] },
  :d4 => { pitches: [:e4, :d4, :c4, :f4, :c#4],     weights: [3, 3, 2, 2, 1] },
  :g4 => { pitches: [:f4, :f#4, :e4, :g4, :a4],     weights: [3, 3, 2, 2, 1] },
  :c4 => { pitches: [:b3, :d4, :c4, :e4, :c#4],     weights: [3, 3, 2, 2, 1] },
}

live_loop :insistent do
  current ||= :e4
  entry = insistent_transitions[current] || insistent_transitions[:e4]

  play current,
    sustain: rrand(0.2, 0.6),
    amp:     rrand(0.55, 0.85)

  sleep rrand(0.15, 0.4)  # tight timing — this character doesn't pause

  current = entry[:pitches].wrand(entry[:weights])
end
```

Now a contrasting character — sparse and patient, pentatonic, comfortable with long rests:

```ruby
# A sparse, patient character — pentatonic, rests frequently, long decays
sparse_transitions = {
  :c5 => { pitches: [:c5, :e5, :e5, :r,  :g5],   weights: [2, 3, 2, 4, 1] },
  :e5 => { pitches: [:c5, :e5, :g5, :r,  :a5],   weights: [3, 2, 3, 3, 1] },
  :g5 => { pitches: [:e5, :g5, :a5, :r,  :c5],   weights: [3, 2, 2, 4, 1] },
  :a5 => { pitches: [:g5, :e5, :a5, :r,  :c6],   weights: [3, 2, 2, 3, 1] },
  :c6 => { pitches: [:a5, :g5, :c5, :r,  :e5],   weights: [3, 3, 2, 3, 1] },
  :r  => { pitches: [:c5, :e5, :g5, :r,  :a5],   weights: [2, 2, 2, 3, 1] },  # from rest, lean toward rest again
}

live_loop :sparse do
  current ||= :c5
  entry = sparse_transitions[current] || sparse_transitions[:c5]

  unless current == :r
    play current,
      attack:  0.01,
      sustain: rrand(0.1, 0.4),
      release: rrand(1.5, 4.0),  # long decay — the note lingers
      amp:     rrand(0.2, 0.5)
  end

  sleep rrand(1.0, 3.5)  # generous gaps between events

  current = entry[:pitches].wrand(entry[:weights])
end
```

Run these simultaneously and listen to the relationship. The insistent voice and the sparse voice occupy different time densities and different registers. Where the insistent voice rarely pauses, the sparse voice rests as often as it sounds. Where the insistent voice pressures by half-steps, the sparse voice moves by thirds and fourths through a pentatonic field.

## Modulating Weights Over Time

Markov weights don't have to be static. They can change as the piece evolves:

```ruby
# Weight modifier: increases over time, making the voice progressively denser
$density = 1.0

live_loop :density_driver do
  $density = [$density + 0.02, 3.0].min  # ramp from 1.0 to 3.0 over time
  sleep 10
end

live_loop :evolving_voice do
  current ||= :c4
  base_weights = [3, 2, 2, 1, 1]

  # Scale all weights by $density — denser as time passes
  scaled = base_weights.map { |w| (w * $density).round.clamp(1, 20) }
  current = [:c4, :d4, :e4, :g4, :a4].wrand(scaled)

  play current, sustain: rrand(0.3, 0.8) / $density, amp: 0.6
  sleep rrand(0.5, 1.5) / $density
end
```

This voice becomes progressively busier as `$density` increases — same probability shape, compressed timescale. This is a continuous parameter driving structural change without any manual intervention.

## Using `ring` for Quasi-Markov Patterns

For characters who need more deliberate patterning rather than pure randomness, `ring` provides cycling through a sequence with controlled variation:

```ruby
# A repeating phrase with slight variation each cycle
base_phrase = ring(:c4, :e4, :g4, :a4, :g4, :e4)

live_loop :quasi_markov do
  pitch = base_phrase.tick

  # Add occasional chromatic neighbour note instead of the expected pitch
  pitch = pitch + [-1, 0, 0, 0, 1].choose if one_in(5)

  play pitch, sustain: 0.8, amp: rrand(0.5, 0.75)
  sleep 0.75
end
```

This produces a recognisable repeating shape with occasional chromatic inflections — more structured than pure Markov, more varied than a fixed sequence.

*Project connection: A programmatic electronic work that runs for 90 minutes requires every character voice to maintain interest and aliveness across that duration without live intervention. Markov chains, modulating weights, and event-triggered table shifts are the three tools that make this possible. Together they produce behaviour that sounds composed but never sounds exactly the same twice.*
